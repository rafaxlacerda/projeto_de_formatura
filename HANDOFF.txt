4. ARCHITECTURE — MONTE CARLO PIPELINE
================================================================================

The Monte Carlo pipeline (in 1.4_geracao_de_cenarios/) works as follows:

1. sortear_monte_carlo.py: generates N realizations per penetration level.
   - Randomly places PVSystem and Storage elements on consumer buses.
   - Assigns irradiance profiles and load uncertainty factors.
   - Saves CSVs under resultados_monte_carlo/realizacoes_sorteadas/pen_{PPP}pct/.

2. simulacao_opendss.py: runs each realization through OpenDSS.
   - For each hour 0-23: sets Load element kW/kvar, calls Solve(), records voltages.

3. controle_primario.py: applies VoltVar control on top of step 2.
   - Loops up to MAX_VOLTVAR_ITER times per hour.
   - Updates Q injected by each DER based on bus voltage.
   - Freezes regulator taps (ControlMode=OFF) during VoltVar iterations.

4. analise_opendss/ contains the post-processed results (tensoes_opendss_completas.csv).

The diagnostic script 5_primario_integrado.py replicates step 2+3 for a SINGLE
realization and exposes internal state (taps, Q per DER, violations per bus, etc.)
that the batch pipeline does not record.


================================================================================
5. ARCHITECTURE — PRIMARY CONTROL (CONTROLE PRIMÁRIO)
================================================================================

Standard: ABNT NBR 16149:2013, §4.7.3

Dead band: [0.97, 1.03] p.u.  → VoltVar outputs Q = 0 in this range
Saturation: [0.90, 1.10] p.u. → VoltVar outputs ±Q_MAX in this range
Q_MAX_PU = 0.4358 = sin(arccos(0.90))

Between dead band and saturation: linear interpolation.

Q sign:
  - V too LOW  (< 0.97) → inject capacitive Q (Q > 0 in generator convention)
  - V too HIGH (> 1.03) → absorb inductive Q (Q < 0 in generator convention)

Maximum Q per DER:
  Q_kvar_max = Q_MAX_PU × |P_inst_kw|

Since Q ≤ Q_MAX_PU × P_inst is always satisfied by construction,
the apparent power increase ΔS = √(P² + Q²) - P ≤ P × (√(1 + Q_MAX_PU²) - 1) ≈ 9.1% × P
is automatically bounded. No P curtailment occurs — OpenDSS Load elements do NOT
have a kVA capacity limit; P is maintained at P_inst regardless of Q dispatch.

This means: DO NOT implement "curtailed energy" or "curtailed P" metrics.
The model does not curtail P. (This was verified and confirmed in session.)

Regulator freeze:
  During VoltVar iterations, ControlMode is set to OFF to freeze LTC/RegControl taps.
  This is the "primary control" assumption: Q is faster than tap-changers.
  After VoltVar converges, ControlMode=STATIC is restored.

6f. MAIN LOOP LOGIC (lines 663-901)
-------------------------------------

main():
  1. redirecionar_circuito(BASE_DSS)
  2. cargas_base = carregar_cargas_base()
  3. pv_real, bess_real, fatores_irradiancia, fatores_carga =
         preparar_cenario_load(PEN_PCT, ID_REALIZACAO)
  4. multiplicadores_perfil_carga = obter_multiplicadores_shapes()
  5. reg_names = dss.RegControls.AllNames()
  6. plot_volt_var_curve(); plot_nominal_power(pv_real)

  Initialize accumulators:
    records_hora, records_der, records_bus, records_taps  (for CSV export)
    viols_por_hora      = []   # list of (viols_pre_dict, viols_pos_dict) — filtered buses
    viols_all_por_hora  = []   # idem — ALL buses

  For each hora in 0..23:
    a. Set cargas_base kW/kvar using LoadShape × uncertainty factor
    b. Set PV kW = -P_nom × irradiance[hora], kvar = 0
    c. Set BESS kW = -P_nom × BESS_PERFIL[hora], kvar = 0
    d. ControlMode=STATIC → Solve()
       (DSSException 485 = "Max Control Iterations" → skip hour)
    e. taps_pre = capture_taps(reg_names)
    f. tensoes_pre = extrair_tensoes_por_barra()
    g. tensoes_pre_all = extrair_tensoes_all_buses()
    h. n_viols_pre, viols_pre = count_voltage_violations(tensoes_pre)
    i. q_refs_pv = _calcular_q_refs(kw_pv, tensoes_pre)
       q_refs_bess = _calcular_q_refs(kw_bess, tensoes_pre)
    j. voltvar_ativo = any(|q| > 1e-6)

    if voltvar_ativo:
      ControlMode=OFF (freeze taps)
      for iter in 0..MAX_VOLTVAR_ITER:
        q_aplicado_pv = dict(q_refs_pv)
        q_aplicado_bess = dict(q_refs_bess)
        editar_load(nome, kw, -q_refs[nome])  # note: negation for load convention
        Solve()
        tensoes_final = extrair_tensoes_por_barra()
        recalculate q_refs
        if all |q| < 1e-6: break (dead band)
      ControlMode=STATIC

    k. taps_pos = capture_taps(reg_names)
    l. tensoes_final_all = extrair_tensoes_all_buses()
    m. n_viols_pos, viols_pos = count_voltage_violations(tensoes_final)
       _, viols_pre_all = count_voltage_violations(tensoes_pre_all)
       _, viols_pos_all = count_voltage_violations(tensoes_final_all)

    n. Append to records_hora, records_der, records_taps, records_bus
    o. Append (viols_pre, viols_pos) to viols_por_hora
    o. Append (viols_pre_all, viols_pos_all) to viols_all_por_hora

  After loop:
    Export CSVs (df_hora, df_der, df_taps)
    plot_taps_por_hora, plot_taps_comparacao
    plot_q_activation, plot_voltvar_efeito
    n_viols_mc = carregar_violacoes_mc(PEN_PCT, ID_REALIZACAO)
    plot_violations(df_hora, n_viols_mc, viols_por_hora)
    plot_violations_all_buses(df_hora, viols_all_por_hora)
    plot_all_bus_vmin, plot_all_bus_vmax

7. CRITICAL DOMAIN KNOWLEDGE
================================================================================

[A] BARRAS_EXCLUIDAS_ANALISE are NOT absent from violations — they CAN have violations.
    They are explicitly EXCLUDED from analysis because they are non-consumer buses.
    Buses 812, 814, 852 in particular DO have voltage violations but are excluded.

[B] Q_MAX_PU = 0.4358 = sin(arccos(0.90)).
    Code comment in controle_primario.py incorrectly says tan(arccos(0.90)).
    tan(arccos(0.90)) ≈ 0.4843. The actual constant 0.4358 is correct; comment is wrong.

[C] Load element sign convention:
    - editar_load(nome, kw, kvar): kW < 0 means INJECTION (generator behavior)
    - kvar > 0 means ABSORPTION in Load convention
    - volt_var_curve returns Q in GENERATOR convention (Q > 0 = capacitive)
    - Therefore: editar_load(nome, kw, -q_refs[nome])  ← negation is MANDATORY

[D] P is NOT curtailed by VoltVar. OpenDSS Load elements have no kVA limit.
    When Q is added, P stays at P_inst. The apparent power increase is:
    ΔS = √(P_inst² + Q²) - P_inst ≤ P_inst × (√(1 + Q_MAX_PU²) - 1) ≈ 9.1% × P_inst
    This is bounded automatically. Do NOT implement curtailed-energy metrics.
    (plot_curtailed_energy was removed from the script — do not re-add it.)

[E] DSSException error code 485 = "Max Control Iterations exceeded".
    Caught by try/except in both the main solve and the VoltVar loop.
    Hours where this occurs are SKIPPED (continue in the for loop).

[F] ControlMode=OFF during VoltVar iterations:
    This freezes LTC (Load Tap Changer) and RegControl operations.
    Primary control assumption: Q response is faster than mechanical tap changes.
    After VoltVar loop: ControlMode=STATIC is restored.

[G] extrair_tensoes_por_barra() (from simulacao_opendss.py) uses BARRAS_EXCLUIDAS_ANALISE.
    extrair_tensoes_all_buses() (local function in 5_primario_integrado.py) does NOT filter.
    Both return {bus_name_lower: [phase_pu_values]}.

[H] BESS_PERFIL:
    hours 10-14: -1.0 (charging/absorption)
    hours 18-21: +1.0 (discharging/injection)
    all others:   0.0 (idle)

[I] DER naming convention in 5_primario_integrado.py:
    PV:   f"PV_{ID_REALIZACAO}_{idx}_barra{linha['barra']}"
    BESS: f"BESS_{ID_REALIZACAO}_{idx}_barra{linha['barra']}"
    (idx = enumeration index over the pv_real / bess_real DataFrame)

[J] The MC CSV (tensoes_opendss_completas.csv) contains ALL buses.
    carregar_violacoes_mc() applies BARRAS_EXCLUIDAS_ANALISE filter to get
    consumer-bus-only counts. This is why fig_violations_consumer_buses.png
    shows lower MC counts than fig_violations_all_buses.png would.
